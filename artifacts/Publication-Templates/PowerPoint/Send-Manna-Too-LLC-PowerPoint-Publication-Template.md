# Send Manna Too LLC PowerPoint Publication Template

## Document Control

Document Status: Planned
Release: 0.3
Artifact Type: PowerPoint Publication Template Specification
Canonical Source: Markdown in Git
Publication Output: PowerPoint presentation format

## Purpose

This artifact defines the controlled structure and formatting requirements for the Send Manna Too LLC PowerPoint publication template.

The PowerPoint template supports executive-ready presentation of approved governance framework content while preserving Markdown in Git as the authoritative source.

## Governance Rule

The PowerPoint publication template is a controlled publication artifact. It does not replace Markdown in Git as the authoritative source for controlled governance framework content.

PowerPoint outputs should summarize, present, or communicate approved content without changing the underlying requirement, decision, or approved meaning.

## Approved Branding Asset

Approved Release 0.3 branding asset:

artifacts/Publication-Templates/Branding/SMT-V2-Horizontal-Premium-Locked.png

Approved use in PowerPoint outputs:

1. Small upper-left brand placement on title slides where technically supported.
2. Small upper-left brand placement on standard content slides where technically supported.
3. Executive-ready presentation branding that does not dominate the slide.
4. PDF outputs generated from approved PowerPoint publication artifacts.

The logo should be scaled down and anchored in the upper-left area. It should resemble a professional webpage header, letterhead, or executive briefing style.

## Required PowerPoint Template Structure

The PowerPoint publication template should support the following slide types:

1. Title slide.
2. Agenda slide.
3. Executive summary slide.
4. Section divider slide.
5. Standard content slide.
6. Table or control matrix slide.
7. Decision or recommendation slide.
8. Closing slide.

## Title Slide Requirements

The title slide should include:

1. Send Manna Too LLC branding using smaller upper-left placement of the approved Release 0.3 branding asset where appropriate.
2. Presentation title.
3. Presentation subtitle or purpose statement where needed.
4. Version or release identifier.
5. Document status where needed.
6. Publication date or release date where applicable.
7. Property and Confidential marking where technically supported.

## Standard Slide Requirements

Standard content slides should support:

1. Small upper-left Send Manna Too LLC branding or firm identification where technically supported.
2. Clear slide title.
3. Concise body content.
4. Structured bullets only where useful.
5. Tables or matrices where needed.
6. Footer marking where technically supported.
7. Slide numbering where technically supported.

## Footer Requirements

The PowerPoint footer should support:

1. Property and Confidential marking where technically supported.
2. Slide numbering where technically supported.
3. Consistent firm identification or approved footer text where technically supported.

## Style Requirements

The PowerPoint template should support consistent styles for:

1. Presentation title.
2. Slide titles.
3. Section dividers.
4. Body text.
5. Bullets.
6. Tables.
7. Callout boxes.
8. Footer text.

## Traceability Requirements

Each PowerPoint publication output should be traceable to:

1. Approved Markdown source or approved release context.
2. Version or publication date where applicable.
3. Source content owner or accountable function where applicable.

## Validation Requirements

Before a PowerPoint publication artifact is approved, confirm:

1. The artifact uses approved Send Manna Too LLC branding or firm identification.
2. Property and Confidential appears where technically supported.
3. Slide numbering appears where technically supported.
4. The artifact is traceable to approved Markdown source or release context.
5. The artifact does not replace Markdown in Git as the authoritative source.
6. The artifact contains no unresolved placeholder status markers.
7. The artifact contains no unintended client data.
8. The artifact is executive-ready, readable, and consistently formatted.
9. The artifact does not change the approved meaning of controlled source content.
