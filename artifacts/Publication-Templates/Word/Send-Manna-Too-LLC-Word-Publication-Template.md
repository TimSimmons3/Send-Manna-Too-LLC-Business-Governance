# Send Manna Too LLC Word Publication Template

## Document Control

Document Status: Planned
Release: 0.3
Artifact Type: Word Publication Template Specification
Canonical Source: Markdown in Git
Publication Output: Word document format

## Purpose

This artifact defines the controlled structure and formatting requirements for the Send Manna Too LLC Word publication template.

The Word template supports executive-ready publication of approved governance framework content while preserving Markdown in Git as the authoritative source.

## Governance Rule

The Word publication template is a controlled publication artifact. It does not replace Markdown in Git as the authoritative source for controlled governance framework content.

Any correction to controlled governance content must be made in the approved Markdown source before regenerating or updating the Word publication output.

## Approved Branding Asset

Approved Release 0.3 branding asset:

artifacts/Publication-Templates/Branding/SMT-V2-Horizontal-Premium-Locked.png

Approved use in Word outputs:

1. Small upper-left brand placement on the cover page.
2. Small upper-left brand placement on standard document pages where technically supported.
3. Letterhead-style or webpage-header-style presentation.
4. Executive-ready business document presentation.

The logo should be scaled down and anchored in the upper-left area. It should not dominate the page or appear as an oversized cover-only graphic.

## Required Word Template Structure

The Word publication template should support the following structure:

1. Cover page.
2. Document control section.
3. Purpose or executive summary section.
4. Main body content sections.
5. Tables, checklists, or control matrices where needed.
6. Approval or revision history section where needed.
7. Appendix section where needed.

## Cover Page Requirements

The cover page should include:

1. Send Manna Too LLC branding using a smaller upper-left placement of the approved Release 0.3 branding asset where appropriate.
2. Document title.
3. Document type.
4. Version or release identifier.
5. Document status.
6. Owner or accountable function.
7. Publication date or release date where applicable.
8. Property and Confidential marking.

## Header Requirements

The Word header should support:

1. Small upper-left Send Manna Too LLC branding or firm identification where technically supported.
2. Short document title where appropriate.
3. No uncontrolled content that could conflict with the canonical Markdown source.

## Footer Requirements

The Word footer should support:

1. Property and Confidential marking.
2. Page numbering in the format Page 1 of X Pages where technically supported.
3. Consistent firm identification or approved footer text.

## Style Requirements

The Word template should support consistent styles for:

1. Title.
2. Subtitle.
3. Heading 1.
4. Heading 2.
5. Heading 3.
6. Body text.
7. Numbered lists.
8. Tables.
9. Notes.
10. Footer text.

## Traceability Requirements

Each Word publication output should be traceable to:

1. Approved Markdown source.
2. Release identifier or approved change context.
3. Version or publication date where applicable.

## Validation Requirements

Before a Word publication artifact is approved, confirm:

1. The artifact uses approved Send Manna Too LLC branding or firm identification.
2. Property and Confidential appears where technically supported.
3. Page numbering follows Page 1 of X Pages where technically supported.
4. The artifact is traceable to approved Markdown source or release context.
5. The artifact does not replace Markdown in Git as the authoritative source.
6. The artifact contains no unresolved placeholder status markers.
7. The artifact contains no unintended client data.
8. The artifact is executive-ready, readable, and consistently formatted.
