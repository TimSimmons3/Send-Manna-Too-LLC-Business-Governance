# Send Manna Too LLC PDF Export Expectations

## Document Control

Document Status: Planned
Release: 0.3
Artifact Type: PDF Export Expectations
Canonical Source: Markdown in Git
Publication Output: PDF final-form format

## Purpose

This artifact defines the controlled expectations for exporting Send Manna Too LLC Business Governance Framework publication artifacts to PDF.

PDF outputs are final-form publication artifacts. They do not replace Markdown in Git as the authoritative source for controlled governance framework content.

## Governance Rule

PDF outputs must be generated from approved source content or approved publication artifacts.

If a PDF requires correction, the correction should be made in the approved source or controlled publication artifact source, then the PDF should be regenerated.

Uncontrolled manual PDF edits should be avoided unless an exception is documented.

## Approved PDF Inputs

Controlled PDF outputs may be generated from:

1. Approved Markdown source rendered through a controlled process.
2. Approved Word publication artifacts generated from controlled source content.
3. Approved PowerPoint publication artifacts generated from controlled source content.
4. Other approved publication artifacts if a decision record authorizes the source.

## Required PDF Elements

A controlled PDF output should include:

1. Send Manna Too LLC branding or firm identification where technically supported.
2. Property and Confidential marking where technically supported.
3. Page numbering in the format Page 1 of X Pages where technically supported.
4. Clear document title or presentation title.
5. Version, release, or publication context where applicable.
6. Traceability to approved source or approved release context.

## Export Expectations

PDF exports should preserve:

1. Layout integrity.
2. Headers and footers where technically supported.
3. Page or slide numbering where technically supported.
4. Readable fonts.
5. Clear tables and matrices.
6. Branding and confidentiality markings.
7. Hyperlinks where technically supported and appropriate.

## Validation Requirements

Before a PDF publication output is approved, confirm:

1. The PDF opens successfully.
2. The PDF was generated from an approved source or approved publication artifact.
3. The PDF content matches the approved source or approved publication artifact.
4. Headers and footers render correctly where technically supported.
5. Property and Confidential appears where technically supported.
6. Page numbering appears where technically supported.
7. Branding or firm identification appears where technically supported.
8. The PDF contains no unresolved placeholder status markers.
9. The PDF contains no unintended client data.
10. The PDF does not replace Markdown in Git as the authoritative source.

## File Naming Expectations

PDF file names should be clear, stable, and traceable.

Recommended file-name elements include:

1. Firm or framework identifier.
2. Document or presentation title.
3. Release or version identifier where applicable.
4. Publication date where useful.

## Release 0.3 Limitation

Release 0.3 defines PDF export expectations but does not require every planned Word or PowerPoint artifact to have an approved PDF export before release closeout.

Any PDF export produced during Release 0.3 must pass the applicable validation controls before being treated as approved.
