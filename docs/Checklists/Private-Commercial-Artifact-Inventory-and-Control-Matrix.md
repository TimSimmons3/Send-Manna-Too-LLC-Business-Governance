## 1. Purpose

This matrix defines the minimum public control fields and validation requirements for inventorying private commercial artifacts without disclosing their confidential substance.

It supports:

1. `OW-0022` private commercial operations and confidential-record controls.
2. `OW-0024` confidential internal rate-card controls.
3. `OW-0025` cost, effort, margin, contingency, and price-boundary model controls.
4. `OW-0026` pricing-exception, strategic-investment, nonprofit, and pro bono controls.
5. `OW-0027` invoicing, payment, expense, travel, remobilization, collection, and bad-debt controls.
6. `OW-0028` commercial scenario, capacity, and KPI controls.
7. `OW-0029` approval history, preservation, backup, recovery, and supersession controls.

The public repository shall contain only the unpopulated control structure and sanitized validation conclusions.

The populated artifact inventory remains private.

## 2. Scope

This matrix applies to:

1. Private commercial repository control evidence.
2. Confidential rate cards.
3. Cost, effort, margin, contingency, and price-boundary models.
4. Pricing exception and strategic-investment records.
5. Nonprofit and pro bono approval records.
6. Invoicing and payment administration records.
7. Expense, travel, and remobilization records.

14. The populated private commercial artifact inventory.

This matrix does not create a private repository, populate a private artifact, establish G2 or G3 completion, authorize external activity, or create retention or disposition authority.

## 4. Use Instructions

For each private artifact family:

1. Use the stable public evidence identifier defined in this matrix.
2. Maintain the populated control record only in the approved private authoritative source.
3. Do not put the private file name, path, party, value, signature, contract term, privileged advice, credential, or recovery detail in public Git.
4. Complete the applicable control fields before approval or reliance.
5. Mark unavailable or pending evidence explicitly.
6. Apply G2 and G3 restrictions.
7. Preserve approval and version history.
8. Validate backup and representative recovery.
9. Record exceptions, incidents, corrective actions, and residual risks.
10. Publish only a sanitized status when authorized.
11. Re-review after a material change.
12. Do not dispose of records unless authorized by a later approved control.

7. Scope.

13. Preservation requirement.
14. Closure condition.

An exception shall not:

1. Put private values in public Git.

3. Bypass required qualified review.

5. Remove preservation requirements.
6. Create unauthorized external authority.
7. Authorize Release 2.7 work.

## 33. Stop Conditions

Stop the affected activity when:

1. G2 has not passed before confidential population or reliance.
2. Applicable G3 requirements are incomplete before private artifact population, approval, or affected live use.
3. The private authoritative source is not identified.
4. Access is excessive or uncontrolled.
5. Encryption is insufficient.
6. Backup is missing.
7. Recovery fails.
8. Version or approval evidence is missing.
9. Integrity cannot be demonstrated.
10. A model cannot reconcile.
11. A private value is proposed for public Git.
12. A public reference reveals protected substance.
13. Qualified review is required but bypassed.
14. A stale version may be used.
15. Preservation requirements are unclear.
16. An incident is unresolved.
17. An exception lacks authority or compensating control.
18. External activity is proposed without authority.
19. Release 2.7 or later work begins.
20. A stricter requirement conflicts with this matrix.

## 34. Fallbacks

| Condition | Required Fallback |
| --- | --- |
| Private source unavailable | Maintain only public controls and unpopulated structures |
| G2 fails | Repair or replace the source and repeat validation |
| G3 remains incomplete | Restrict affected population, approval, conclusion, and use |
| Qualified review unavailable | Document status and defer affected decision |
| Backup fails | Repair backup and revalidate |
| Recovery fails | Repair recovery and retest |
| Model does not reconcile | Freeze approval, correct, and rerun tests |
| Public summary is unsafe | Aggregate, redact, restrict, delay, or keep private |
| Record state is unclear | Apply the more restrictive state |
| Disposition authority is unclear | Preserve the record |
| Incident occurs | Contain, preserve, remediate, and revalidate |
