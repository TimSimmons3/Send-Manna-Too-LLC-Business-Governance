## Purpose

This document defines the controlled requirements for Word publication templates used by Send Manna Too LLC Business Governance Framework deliverables.

The purpose is to ensure that Word-formatted deliverables are professional, consistent, traceable, and clearly treated as publication outputs rather than authoritative source records.

## Authority

Markdown in Git remains the authoritative source of truth for controlled governance framework content.

Word documents are publication formats only. A Word document must not become the primary record unless a future approved decision changes the governance architecture.

## Scope

This document applies to controlled Word publication templates used for:

- Governance framework documents.
- Client-ready advisory documents.
- Executive-ready reports.
- Formal policies, standards, procedures, and playbooks.
- Other controlled deliverables approved for Word publication.

## Required Template Elements

A controlled Word publication template must include:

- Send Manna Too LLC logo in the header where technically supported.
- Send Manna Too LLC logo or approved firm identification in the footer where technically supported.
- Footer text stating Property and Confidential.
- Page numbering in the format Page 1 of X Pages where technically supported.
- Document title.
- Document owner.
- Version or release identifier.
- Document status.
- Effective date or publication date when applicable.
- Approved classification or confidentiality marking.
- Consistent heading structure.
- Consistent body text style.
- Consistent table style.
- Consistent footer and page-number style.

## Document Control Requirements

Each controlled Word publication output should include document control information sufficient to support review, approval, traceability, and future maintenance.

At minimum, document control information should identify:

- Document title.
- Owner.
- Version or release.
- Status.
- Approval state.
- Canonical source location.
- Publication date when applicable.

## Styling Requirements

Word templates must support a professional and executive-ready format.

Required style expectations:

- Clear title hierarchy.
- Readable body text.
- Consistent spacing.
- Consistent table formatting.
- No decorative elements that reduce readability.
- No uncontrolled branding variations.
- No logos, graphics, or design elements that conflict with approved Send Manna Too LLC branding.

## Header Requirements

The header should support:

- Send Manna Too LLC branding.
- Document title or short title when appropriate.
- Optional release or classification indicator.

The header must not include uncontrolled content that could become outdated or conflict with the canonical Markdown source.

## Footer Requirements

The footer should support:

- Property and Confidential marking.
- Page numbering in the format Page 1 of X Pages where technically supported.
- Optional document version or release identifier.

## Publication Control Requirements

Word publication outputs must be traceable to the canonical Markdown source used to produce them.

At minimum, publication records should preserve:

- Source Markdown file path.
- Git commit or release tag when available.
- Publication date.
- Export method.
- Reviewer or approver when applicable.

## Restrictions

Controlled Word templates must not:

- Replace Markdown in Git as the authoritative source.
- Introduce duplicate governance requirements already controlled elsewhere.
- Include unresolved placeholder status markers in final deliverables.
- Contain client-specific content unless the template is explicitly approved for that purpose.
- Contain confidential client information in the base template.

## Validation Requirements

Before a Word publication template is approved, validate that:

- Required branding elements are present.
- Property and Confidential appears in the footer.
- Page numbering is configured where technically supported.
- Document control fields are present.
- Styles are consistent.
- The template does not contain unintended client data.
- The template does not contain unresolved placeholder status markers.
- The template aligns with the Document Quality Standard.

## Future Implementation Note

This requirements document defines what the controlled Word template must support.

The actual Word template file may be created later as a publication artifact, but the requirements in this document remain controlled in Markdown and Git.
