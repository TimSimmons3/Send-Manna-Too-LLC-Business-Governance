## Purpose

This document defines the controlled requirements for PowerPoint publication templates used by Send Manna Too LLC Business Governance Framework deliverables.

The purpose is to ensure that presentation-format deliverables are professional, consistent, traceable, and clearly treated as publication outputs rather than authoritative source records.

## Authority

Markdown in Git remains the authoritative source of truth for controlled governance framework content.

PowerPoint files are publication and presentation formats only. A PowerPoint file must not become the primary record unless a future approved decision changes the governance architecture.

## Scope

This document applies to controlled PowerPoint publication templates used for:

- Executive presentations.
- Client-ready advisory presentations.
- Governance framework summaries.
- Board-style reporting packages.
- Training or awareness materials based on controlled governance content.
- Other presentation deliverables approved for PowerPoint publication.

## Required Template Elements

A controlled PowerPoint publication template must include:

- Send Manna Too LLC branding where technically supported.
- Title slide layout.
- Section divider slide layout.
- Standard content slide layout.
- Table slide layout.
- Risk or issue summary slide layout.
- Decision or recommendation slide layout.
- Closing slide layout.
- Footer text stating Property and Confidential where technically supported.
- Slide numbering where technically supported.
- Version, release, or publication identifier where appropriate.
- Consistent heading, body, table, and footer styles.

## Branding Requirements

PowerPoint templates must use controlled Send Manna Too LLC branding.

Branding should support:

- Professional executive presentation style.
- Clear firm identification.
- Consistent placement of logo or firm name.
- Consistent footer and slide-number treatment.
- No uncontrolled design variations.

Branding must not reduce readability or interfere with the purpose of the slide.

## Document Control Requirements

Each controlled PowerPoint publication output should include enough information to support traceability and review.

At minimum, document control information should identify:

- Presentation title.
- Owner.
- Version or release.
- Status or approval state when applicable.
- Canonical source location where applicable.
- Publication date when applicable.

## Slide Design Requirements

PowerPoint templates must support concise, executive-ready communication.

Required design expectations:

- Clear slide titles.
- Readable text size.
- Consistent spacing.
- Consistent table formatting.
- Limited visual clutter.
- Professional use of graphics.
- No decorative elements that reduce clarity.
- No uncontrolled client-specific branding in the base template.

## Content Control Requirements

Presentation content derived from controlled governance framework documents must remain aligned with the approved source.

PowerPoint outputs should summarize or present controlled content without changing the underlying requirement, decision, or approved meaning.

If a presentation introduces new governance decisions, the decision must be recorded in the Decision Register before the presentation is treated as approved.

## Publication Control Requirements

PowerPoint publication outputs must be traceable to the approved source or release context used to produce them.

At minimum, publication records should preserve:

- Source Markdown file path when applicable.
- Source Git commit or release tag when available.
- Presentation file path.
- Publication date.
- Reviewer or approver when applicable.

## Restrictions

Controlled PowerPoint templates must not:

- Replace Markdown in Git as the authoritative source.
- Introduce duplicate governance requirements already controlled elsewhere.
- Include unresolved placeholder status markers in final deliverables.
- Contain confidential client information in the base template.
- Include client-specific branding unless the template is explicitly approved for that purpose.
- Use slide design choices that weaken readability or professional quality.

## Validation Requirements

Before a PowerPoint publication template is approved, validate that:

- Required branding elements are present where technically supported.
- Property and Confidential appears where technically supported.
- Slide numbering is configured where technically supported.
- Required slide layouts are present.
- Styles are consistent.
- Text is readable.
- Tables are readable.
- The template does not contain unintended client data.
- The template does not contain unresolved placeholder status markers.
- The template aligns with the Document Quality Standard.

## Future Implementation Note

This requirements document defines what the controlled PowerPoint template must support.

The actual PowerPoint template file may be created later as a publication artifact, but the requirements in this document remain controlled in Markdown and Git.
